Expanded · Spacer · SizedBox
space controlRow/Column only
Expanded fills all remaining space. Spacer is invisible Expanded that pushes. SizedBox is a fixed gap or box.
  • Expanded(flex:2) vs (flex:1) → 2:1 ratio
  • Spacer() between items → pushes to opposite ends
  • SizedBox(height:16) → 16px vertical gap
  • SizedBox.shrink() → zero-space placeholder

ListView.builder
lazyrepeating
Any repeating same-shape items. builder() only renders visible items — critical for 50+ items. Never use ListView() with children list for dynamic data.
  • ListView() → max 10 static children
  • ListView.builder → always for dynamic data
  • ListView.separated → with Divider between
  • shrinkWrap:true + NeverScrollable → inside Column
ListView.builder itemCount: flights.length itemBuilder: (ctx, i) => FlightCard(flights[i]) └── Row ├── AirlineLogo ├── Expanded → Column └── PriceText
lazy list

// with dividers

GridView.builder
grid2D lazy
When ListView items have equal visual weight and a 2-column or more layout makes sense — shop, gallery, categories, amenities.

Slivers — CustomScrollView
Header + sticky chips + scrolling list all moving as one. The pattern behind every serious SRP.
SliverAppBar
Collapsing header. expandedHeight = hero size. pinned:true → toolbar stays. floating:true → reappears on scroll-up. Put search bar in flexibleSpace → FlexibleSpaceBar.
SliverToBoxAdapter
Bridge widget. Normal widgets can't go directly in slivers — wrap them here. Promo banners, section headers, any one-off widget between lists.
SliverPersistentHeader (pinned)
Sticky widget. pinned:true → sticks to top while rest scrolls. Filter chips, tabs, "Today/Yesterday" section headers. Needs delegate with minExtent + maxExtent.
SliverList / SliverGrid
Lazy list. Like ListView.builder but inside CustomScrollView. All above slivers scroll away together as one unified scroll system.

CustomScrollView
sliversunified scroll
Search header collapses. Filter chips stay pinned. Flight cards scroll underneath. All as ONE unified scroll — impossible with Column + ListView.
  • SliverAppBar(pinned:true) → toolbar stays visible
  • SliverPersistentHeader(pinned:true) → chips stick
  • SliverList → lazy flight cards
  • SliverToBoxAdapter → any normal widget in between
CustomScrollView slivers: [ SliverAppBar └── SearchHeader SliverPersistentHeader └── FilterChips ← sticky SliverList └── FlightCard × n ]
flights SRP
Scaffold(

            background:
              SearchHeader(),
          ),
      ),
      SliverPersistentHeader(
        pinned: true,
        delegate:
          _FilterDelegate(),
      ),
      SliverList(
        delegate:
          SliverChildBuilderDelegate(
            (ctx, i) =>
              FlightCard(flights[i]),
            childCount:
              flights.length,
          ),
      ),
    ],
  ),

PageView
swipe pagesfull screen
ListView = scroll through content. PageView = navigate through pages. Onboarding, stories, image gallery. Dots need onPageChanged + setState.
  • PageController → programmatic navigation
  • viewportFraction:0.85 → peek next (carousel)
  • Axis.vertical → TikTok/Reels style
  • onPageChanged → update dots with setState
Stack ├── PageView.builder │ onPageChanged: update dot │ └── OnboardPage(pages[i]) └── Positioned(bottom) └── Column ├── Row(AnimatedDots) └── ElevatedButton

LayoutBuilder · Wrap
responsiveparent-aware
LayoutBuilder responds to parent container size — better than MediaQuery which only knows screen size. Wrap = Row that breaks to next line.
  • LayoutBuilder → switch Row/Column by width
  • Wrap → filter chips, tags that wrap naturally
  • MediaQuery → screen size, keyboard insets
  • SafeArea → avoid notch + system bars
LayoutBuilder builder: (ctx, constraints) { final wide = constraints.maxWidth > 600; return wide ? Row([Left(), Right()]) : Column([Top(), Bottom()]); }
LayoutBuilder + Wrap
LayoutBuilder(
  builder: (ctx, c) {
    final wide =
      c.maxWidth > 600;
    return wide
      ? Row([Left(), Right()])
      : Column([Top(), Bottom()]);
  },
)
